One grid, every edge.

Meridian isn't a model in a single data center with a CDN in front. It's 142 independent inference clusters running identical weights, synchronized through a low-latency mesh.

Requests route to the nearest region automatically. Context follows the user across nodes without re-establishing state. The model you talk to in Tokyo is the same model you talk to in Toronto — same weights, same behavior, different server.

OpenAI-compatible. Most SDKs work by changing the base URL.

quickstart.sh 
# your first Meridian call
curl https://api.meridian.ai/v1/chat/completions \
  -H "Authorization: Bearer $MERIDIAN_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "meridian-1",
    "messages": [
      { "role": "user", "content": "translate to 50 languages" }
    ]
  }'
